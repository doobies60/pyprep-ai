# 開発実績ポートフォリオ
## 生成AI搭載型 学習支援プラットフォーム
### Python3 基礎演習アカデミー (PyPrep-AI)

---

## 1. プロジェクト概要

**アプリケーション名:** Python3 基礎演習アカデミー (PyPrep-AI)

**開発目的:**
Python 3 エンジニア認定基礎試験の受験者向けに、**効率的かつ持続可能な学習環境**を提供すること。

**解決する課題:**
従来の「固定された過去問学習」における**「問題の枯渇」「飽き」**を解決。

**ソリューション:**
Google Gemini (生成AI) を組み込み、学習進度に合わせて**無限に問題を生成**し、詳細な解説を行うハイブリッド型システムを構築。

---

## 2. 使用技術スタック (Backend/DB)

堅牢性と拡張性を重視した選定を行いました。

| カテゴリ | 技術 | 採用理由・役割 |
| :--- | :--- | :--- |
| **言語** | **Python 3.12** | 最新構文と型ヒントによる堅牢な実装 |
| **FW** | **Flask** | Blueprint機能による機能分割（認証・メイン・管理） |
| **DB** | **SQLite / PostgreSQL** | 環境変数による自動切り替え、SQLAlchemy (ORM) |
| **Auth** | **Flask-Login** | セッション管理、CSRF対策 (Flask-WTF) |

---

## 3. 使用技術スタック (AI/Frontend/Tools)

UX向上と運用効率化のための技術選定です。

| カテゴリ | 技術 | 採用理由・役割 |
| :--- | :--- | :--- |
| **生成AI** | **Google Gemini API** | `google-genai` を用いた高精度な問題・解説生成 |
| **分析** | **Streamlit** | 管理者用ダッシュボード、学習ログの可視化 |
| **Front** | **Jinja2 / HTML / CSS** | サーバーサイドレンダリングによる動的生成 |
| **Dev** | **Git / python-dotenv** | バージョン管理、機密情報の環境変数管理 |

---

## 4. システムアーキテクチャのこだわり

**MVCモデルとモジュラー設計**
大規模開発を見据え、Flaskの **Blueprint** を採用して機能を分割。
*   `auth`: 認証基盤
*   `main`: 学習ロジック・AI連携
*   `admin`: 管理機能

**ハイブリッド出題システム**
*   **基本問題:** ローカルDBから高速に出題（低レイテンシ）
*   **応用問題:** 必要時のみAI APIを非同期呼び出し
*   **効果:** コスト削減とユーザー体験の向上を両立

---

## 5. 技術的挑戦: AIトークン管理

**課題:** APIコスト管理と過剰利用の防止
**解決策:** 独自のトークン消費システムを実装

*   ユーザーごとに「1日100トークン」を付与。
*   `before_request` フックを活用し、日付変更時に**自動リセット**するロジックを実装。
*   ステートレスなWeb通信の中で、DBを用いてユーザー状態を厳格に管理。

---

## 6. 技術的挑戦: 堅牢性と品質保証

**フォールバック機能**
*   AI APIのエラーやタイムアウト発生時、即座に**内部DBの類似問題へ切り替え**。
*   ユーザーにエラー画面を見せないUX設計。

**プロンプトエンジニアリング**
*   AIの回答精度（ハルシネーション抑制）のため、JSONスキーマを含む**厳格なシステムプロンプト**を設計。
*   プログラム側でパースしやすい構造化データを出力。

---

## 7. 管理・運用ツール (Streamlit)

開発者・管理者向けの分析ツールを自作し、データドリブンな改善を実現。

*   **check_db.py:** データベース管理ダッシュボード
*   **機能:**
    *   全問題データの検索・閲覧
    *   章ごとの出題数分布グラフ (Pandas)
    *   難易度バランスの可視化
*   **効果:** 本番リリース後のコンテンツ品質管理を容易に。

---

## 8. 開発を通じて得られたスキル

本プロジェクトを通じて、以下のエンジニアリング能力を高めました。

1.  **フルスタック開発力:**
    DB設計からUI実装、API連携まで一気通貫で実装する力。
2.  **製品視点での設計:**
    「AIの遅延をどう隠すか」「コストをどう抑えるか」といった実運用上の課題解決。
3.  **生成AI活用スキル:**
    API連携だけでなく、意図通りの出力を得るためのプロンプト設計力。
